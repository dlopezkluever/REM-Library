## File Naming Conventions

| Category                | Convention                                      | Example                                |
| ----------------------- | ----------------------------------------------- | -------------------------------------- |
| React components        | PascalCase + `.tsx`                             | `EntityBadge.tsx`, `GraphCanvas.tsx`   |
| Page components         | PascalCase + `Page.tsx`                         | `EntityDetailPage.tsx`                 |
| Hooks                   | camelCase, starts with `use`, `.ts`             | `useGraph.ts`, `useSearch.ts`          |
| API query functions     | camelCase by entity/feature, `.ts`              | `entities.ts`, `claims.ts`             |
| Supabase Edge Functions | kebab-case folder                               | `trigger-transcription/`               |
| Stores                  | camelCase + `Store`, `.ts`                      | `graphStore.ts`                        |
| Types                   | camelCase for files, PascalCase for types       | `domain.ts`, `type EntityNode = {...}` |
| Constants               | camelCase for files, SCREAMING_SNAKE for values | `entityTypes.ts`, `ENTITY_COLORS`      |
| SQL migrations          | `YYYYMMDDHHMMSS_description.sql`                | `20260523120000_initial_schema.sql`    |

## Import & Export Patterns

### Path aliases (configure in `vite.config.ts` and `tsconfig.json`)

```
@/components → src/components
@/pages      → src/pages
@/hooks      → src/hooks
@/lib        → src/lib
@/types      → src/types
@/stores     → src/stores
@/constants  → src/constants
```

### No barrel files

Do not create `index.ts` re-export files inside component subdirectories. Import directly from the file:

```ts
// Correct
import { EntityBadge } from '@/components/entity/EntityBadge'

// Avoid
import { EntityBadge } from '@/components/entity'
```

Barrel files create circular dependency risk and make tree-shaking harder to reason about.

### Export pattern

- Components: named export (not default export). Default exports make refactoring and grep harder.
- Page components: default export (React Router convention).
- Hooks: named export.
- API functions: named exports from their module file.
- Types: named exports.

## Code Style

### TypeScript

- `strict: true` in tsconfig — no exceptions.
- No `any`. Use `unknown` when the type is genuinely unknown, then narrow with a type guard.
- Prefer `interface` for object shapes that describe data; `type` for unions, intersections, and computed types.
- All Supabase query results typed using the generated `Database` type from `types/database.ts`.
- Zod schemas for validating external data (Claude API extraction output, AssemblyAI webhook payloads).

### React

- Functional components only; no class components.
- Arrow function components: `const EntityBadge = ({ type }: Props) => { ... }`.
- Props interfaces defined inline above the component or in the same file — not in a separate types file unless shared.
- `useCallback` and `useMemo` only when there is a measurable performance need — not as a default habit.
- Never read from Zustand stores in deeply nested leaf components; pass data down as props to keep components testable.

### Supabase queries

All database access goes through `src/lib/api/` modules — no direct `supabase.from(...)` calls in components or hooks.

```ts
// src/lib/api/entities.ts
export async function getEntityBySlug(slug: string) {
  const { data, error } = await supabase
    .from('entities')
    .select('*, relationships(*)')
    .eq('slug', slug)
    .eq('status', 'published')
    .single()
  if (error) throw error
  return data
}
```

Components call these functions via TanStack Query hooks:

```ts
const { data: entity } = useQuery({
  queryKey: ['entity', slug],
  queryFn: () => getEntityBySlug(slug),
})
```

### Comments

Write no comments by default. Add a comment only when the WHY is non-obvious:

- A hidden constraint or non-obvious invariant
- A workaround for a specific external bug
- Behavior that would surprise the next reader

Never comment what the code does — well-named identifiers already do that.

## File Length Guideline

Target ≤ 500 lines per file. If a component file grows beyond this:

- Extract sub-components into their own files.
- Extract data-fetching logic into a custom hook.
- Extract complex calculations into a `lib/` utility function.

The 500-line limit is a signal, not a hard rule — a 550-line file is fine if it is genuinely cohesive. A 300-line file with three unrelated concerns should be split.

## Database Schema

Deployed and managed via Supabase migrations in `supabase/migrations/`.

### Core tables

```sql
-- Enums
CREATE TYPE entity_type AS ENUM ('symbol', 'figure', 'narrative', 'culture', 'trope');
CREATE TYPE relationship_type AS ENUM (
  'symbolizes', 'appears_in', 'belongs_to', 'parallels', 'instantiates', 'supports'
);
CREATE TYPE content_status AS ENUM ('draft', 'published', 'archived', 'disputed');
CREATE TYPE source_format AS ENUM ('audio', 'video', 'text', 'book', 'url');
CREATE TYPE source_tier AS ENUM ('primary', 'secondary');
CREATE TYPE pipeline_stage AS ENUM (
  'uploaded', 'transcribing', 'chunking', 'extracting', 'review', 'curated', 'published'
);
CREATE TYPE extraction_status AS ENUM ('pending', 'confirmed', 'edited', 'rejected', 'merged');
CREATE TYPE admin_role AS ENUM ('super_admin', 'editor', 'viewer');

-- Entity nodes (Symbols, Figures, Narratives, Cultures, Tropes)
CREATE TABLE entities (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  type            entity_type NOT NULL,
  name            text NOT NULL,
  slug            text UNIQUE NOT NULL,
  aliases         text[] DEFAULT '{}',
  description     text,
  confidence_score float DEFAULT 0 CHECK (confidence_score BETWEEN 0 AND 1),
  confidence_override float CHECK (confidence_override BETWEEN 0 AND 1),
  position_x      float,   -- ForceAtlas2 persisted coordinates
  position_y      float,
  status          content_status DEFAULT 'draft',
  created_at      timestamptz DEFAULT now(),
  updated_at      timestamptz DEFAULT now(),
  fts             tsvector GENERATED ALWAYS AS (
    to_tsvector('english', name || ' ' || coalesce(description, ''))
  ) STORED
);
CREATE INDEX entities_fts_idx ON entities USING GIN (fts);
CREATE INDEX entities_type_status_idx ON entities (type, status);

-- Relationship edges
CREATE TABLE relationships (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  from_entity_id  uuid REFERENCES entities(id) ON DELETE CASCADE,
  to_entity_id    uuid REFERENCES entities(id) ON DELETE CASCADE,
  type            relationship_type NOT NULL,
  weight          float DEFAULT 1.0,
  claim_ids       uuid[] DEFAULT '{}',
  created_at      timestamptz DEFAULT now()
);
CREATE INDEX relationships_from_idx ON relationships (from_entity_id);
CREATE INDEX relationships_to_idx ON relationships (to_entity_id);

-- Claims (interpretive assertions)
CREATE TABLE claims (
  id                  uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  statement           text NOT NULL,
  detailed_argument   text,
  author_id           uuid REFERENCES profiles(id),
  confidence_score    float DEFAULT 0,
  confidence_override float,
  status              content_status DEFAULT 'draft',
  created_at          timestamptz DEFAULT now(),
  updated_at          timestamptz DEFAULT now()
);

-- Which entities a claim references
CREATE TABLE claim_entities (
  claim_id    uuid REFERENCES claims(id) ON DELETE CASCADE,
  entity_id   uuid REFERENCES entities(id) ON DELETE CASCADE,
  PRIMARY KEY (claim_id, entity_id)
);

-- Sources
CREATE TABLE sources (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  title           text NOT NULL,
  authors         text[] DEFAULT '{}',
  publication_date date,
  format          source_format NOT NULL,
  tier            source_tier NOT NULL,
  url             text,
  file_path       text,        -- Supabase Storage path
  duration_seconds int,
  page_count      int,
  pipeline_stage  pipeline_stage DEFAULT 'uploaded',
  created_at      timestamptz DEFAULT now()
);

-- Precise locations within sources (evidence anchors)
CREATE TABLE source_anchors (
  id                  uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  source_id           uuid REFERENCES sources(id) ON DELETE CASCADE,
  start_timestamp_sec int,     -- for audio/video
  end_timestamp_sec   int,
  start_page          int,     -- for text/books
  end_page            int,
  transcript_excerpt  text,
  speaker             text,
  created_at          timestamptz DEFAULT now()
);

-- Which anchors support a claim
CREATE TABLE claim_evidence (
  claim_id    uuid REFERENCES claims(id) ON DELETE CASCADE,
  anchor_id   uuid REFERENCES source_anchors(id) ON DELETE CASCADE,
  PRIMARY KEY (claim_id, anchor_id)
);

-- Text chunks from sources
CREATE TABLE chunks (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  source_id       uuid REFERENCES sources(id) ON DELETE CASCADE,
  chunk_index     int NOT NULL,
  start_sec       int,
  end_sec         int,
  speaker         text,
  raw_text        text NOT NULL,
  fts             tsvector GENERATED ALWAYS AS (
    to_tsvector('english', raw_text)
  ) STORED
);
CREATE INDEX chunks_source_idx ON chunks (source_id);
CREATE INDEX chunks_fts_idx ON chunks USING GIN (fts);

-- AI extraction results pending review
CREATE TABLE extractions (
  id              uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  chunk_id        uuid REFERENCES chunks(id) ON DELETE CASCADE,
  extraction_data jsonb NOT NULL,  -- structured JSON from Claude
  status          extraction_status DEFAULT 'pending',
  reviewed_by     uuid REFERENCES profiles(id),
  reviewed_at     timestamptz,
  created_at      timestamptz DEFAULT now()
);

-- User profiles and admin roles
CREATE TABLE profiles (
  id          uuid REFERENCES auth.users(id) PRIMARY KEY,
  email       text NOT NULL,
  display_name text,
  role        admin_role DEFAULT 'viewer',
  created_at  timestamptz DEFAULT now()
);
```

### RLS policies

All tables have RLS enabled. Core rules:

- `entities`, `relationships`, `claims`, `source_anchors`, `sources`: public `SELECT` where `status = 'published'`; INSERT/UPDATE/DELETE require `is_admin()` function (checks JWT role claim)
- `extractions`, `chunks`, `profiles`: no public access; require `is_admin()`
- Admin role check function:

```sql
CREATE FUNCTION is_admin() RETURNS boolean AS $$
  SELECT coalesce(
    (SELECT role IN ('super_admin', 'editor')
     FROM profiles WHERE id = auth.uid()),
    false
  );
$$ LANGUAGE sql SECURITY DEFINER;
```

### Access patterns

- Graph load: `SELECT id, type, name, slug, confidence_score, position_x, position_y FROM entities WHERE status = 'published'` + `SELECT * FROM relationships`
- Entity detail: entity by slug + JOINed relationships + claims via `claim_entities` + source_anchors via `claim_evidence`
- 1-hop neighborhood (mini-graph): `SELECT * FROM relationships WHERE from_entity_id = $id OR to_entity_id = $id`
- 2-hop neighborhood: recursive CTE (max depth 2)
- Full-text search: `SELECT * FROM entities WHERE fts @@ websearch_to_tsquery('english', $query)`

```
      graph/buildGraphology.test.ts

      useSearch.test.ts
    components/
      entity/EntityBadge.test.tsx
supabase/
  __tests__/

```

- Unit tests: Vitest for pure functions in `lib/` and `hooks/`
- Component tests: Vitest + React Testing Library for UI components
- Integration tests: Vitest + real Supabase local instance for RLS policies and API functions
- No mocking of Supabase in integration tests — use `supabase start` (local Docker) for a real DB

## Environment Variables

Required in `.env.local` (never committed):

```
VITE_SUPABASE_URL=
VITE_SUPABASE_ANON_KEY=
SUPABASE_SERVICE_ROLE_KEY=   # Only for server-side scripts — never expose to browser
ASSEMBLYAI_API_KEY=
ANTHROPIC_API_KEY=
```

All `VITE_` prefixed variables are exposed to the browser bundle. Variables without `VITE_` are Node.js only (scripts and Edge Functions). Never put `SUPABASE_SERVICE_ROLE_KEY` or `ANTHROPIC_API_KEY` in a `VITE_` variable.
